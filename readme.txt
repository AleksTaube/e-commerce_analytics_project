Данный демо проект посвящен типовой аналитике продаж. 
Анализ существующих датафреймов решает следующие задачи:

1. Поиск количества пользователей, которые совершили покупку только один раз.
2. Поиск среднего количества заказов в месяц, которое не доставляется по разным причинам (выяивить причины, вывести детализацию по ним).
3. Определение дня недели, который каждый товар чаще всего покупается.
4. Определение среднего количества покупок в неделю у каждого пользователя (в расрпеделении по месяцам).
5. Когортный анализ пользователей. Определение retention rate.
6. RFM-сегментация пользователей. (R - время от последней покупки пользователя до текущей даты, F - суммарное количество покупок у пользователя за всё время, M - сумма покупок за всё время).

Анализ проводился с использованием Python (jupiter notebook). Каждая задача разобрана в отельном .ipynb файле с комментариями.
Используемые датафреймы подгружаются из яндекс.диска непосредственно в скриптах, а также приложены в данном репозитории.
